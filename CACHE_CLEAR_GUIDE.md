# Руководство по очистке кеша Yii2

## Проблема
При внесении изменений в CSS/JS файлы, они не отображаются на сайте из-за кеширования Yii2.

## Решение

### Способ 1: Автоматический скрипт (Рекомендуется)
```bash
./clear-cache.sh
```

### Способ 2: Ручная очистка
```bash
# Очистка assets
rm -rf web/assets/*
touch web/assets/.gitkeep

# Очистка runtime cache
rm -rf runtime/cache/*

# Очистка runtime debug
rm -rf runtime/debug/*
```

### Способ 3: Через браузер
После очистки кеша на сервере:
- **Chrome/Firefox**: `Ctrl + Shift + R` (Windows/Linux) или `Cmd + Shift + R` (Mac)
- **Safari**: `Cmd + Option + R`

## Настройки кеширования

В `config/web.php` уже включено автоматическое добавление timestamp к assets:
```php
'assetManager' => [
    'appendTimestamp' => true,  // Автоматическое версионирование
    'linkAssets' => true,
],
```

## Когда нужно очищать кеш?

1. ✅ После изменения CSS файлов
2. ✅ После изменения JS файлов
3. ✅ После обновления Asset классов
4. ✅ После изменения конфигурации
5. ✅ При странном поведении сайта

## Проверка применения изменений

1. Откройте DevTools (F12)
2. Перейдите на вкладку Network
3. Обновите страницу
4. Проверьте, что CSS файлы загружаются с новым timestamp
   - Пример: `product.css?v=1699301234`

## Важно!

- Всегда очищайте кеш после внесения изменений в CSS/JS
- На production сервере кеш очищается автоматически при деплое
- В режиме разработки (YII_DEBUG=true) кеширование минимально
